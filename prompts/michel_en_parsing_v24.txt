I am uploading a PDF document containing pages from an English-language academic publication about Old Assyrian/Akkadian legal texts. I need you to perform a structured, semantic data extraction. It is critical to read the pages in order.

**ROLE:**
You are an expert Assyriologist and Data Engineer. You are processing an English academic chapter/book that contains Old Assyrian/Akkadian tablet transliterations with English translations embedded within scholarly commentary.

**CRITICAL TRIGGER (Read First):**
Locate the texts by finding **Tablet Edition Blocks** within the academic paper.
* **Transliteration blocks:** Look for blocks of numbered Old Assyrian/Akkadian transliteration lines. These are recognizable by: sequential line numbers (1, 2, 3, ...), Akkadian words with hyphens and special characters (š, ṣ, ṭ, ḫ, subscripts like ₄, ₆), Sumerograms (KÙ.BABBAR, DUMU, GÍN, KIŠIB), and surface markers (Obv., Rev., Lo.e., Le.e., Up.e., U.e., R.e.). Tablet headers vary widely: a bold number with title (e.g., "**69.** *Letter. Buzazu, ill, did not send copper*"), a collection number (BM 113394, AO 9337, NBC 1677), or a publication reference.
* **Translation sections:** English translations typically appear as a paragraph below the transliteration block, using superscript line-range references (e.g., `¹⁻²Say to Puzur-Aššur...`, `²⁻⁵Concerning...`, `¹²⁻¹⁴"Give me the copper..."`). The translation may sometimes appear before the transliteration.
* **Envelope + Tablet:** Some texts have both an *Envelope* and a *Tablet* section, each with their own transliteration and translation. Treat these as **separate** extraction blocks with independent line numbering. Use the same tablet ID for both but append the section label to distinguish (e.g., ID "NBC 1846 Envelope" and "NBC 1846 Tablet").
* **Ignore:** Title pages, abstracts, introductions, section headers (e.g., "For a few talents of copper..."), bibliographic commentary between texts, footnotes, philological notes (e.g., "l. 7: The Assyrians refer to..."), photograph plates/figures, and any pages with no transliteration blocks. If a page has no relevant content, return `[]`.

**EXTRACTION INSTRUCTIONS:**
Your goal is to extract the text by matching the MEANING of the Akkadian to the MEANING of the English translation.

1.  **Identify the Page Number & ID:** Track the PDF page number. For the ID, use the **collection number** (e.g., "BM 113394", "AO 9337", "NBC 1677", "VAT 13515", "Gi 1-5") as the primary identifier. If the collection number is "unknown", use the text number (e.g., "79") instead. If a text continues onto a new page, use the ID from the previous page.
2.  **Semantic Chunking (Maximum Atomicity):** Read the Akkadian transliteration block and the associated English translation. Break the text down into the smallest, most "atomic" chunks possible.
    * Use the superscript line-range references in the translation (e.g., `¹⁻²`, `³⁻⁵`, `²¹⁻²⁵`) as your initial guide to create chunks.
    * If the translation is a continuous paragraph without line references, align chunks by matching the meaning of the Akkadian to the meaning of the English.
    * **The Atomic List Rule:** You MUST make the data as atomic as possible. If a single translation range contains a list of distinct seals (KIŠIB), witnesses, people, or itemized goods, **break that chunk down further** into distinct, individual JSON objects for each person/item.
3.  **Calculate Line Ranges:** For each chunk, output the clean integer values in the `start_line` and `end_line` fields. Use the numbered lines in the transliteration block (1, 2, 3, ...) directly. Surface markers (Obv., Rev., Lo.e., Le.e., Up.e., U.e., R.e.) indicate tablet surface sections but the line numbers are sequential across them — use the integer line number as-is. For "Obv." treat it as line 1 if no explicit number is given.
4.  **Language:** Set `"language"` to `"en"`.
5.  **Handle Unmatched Text:** If Akkadian text is transliterated but has no corresponding English translation (or the translation section is absent for that tablet), extract it as its own chunk and set `"translation": "MISSING"`.
6.  **Assess Quality Metrics:**
    * **`ocr_quality`**: Rate image clarity ("high", "medium", "low"). Born-digital PDFs are usually "high". Lower only for scanned/degraded pages.
    * **`match_quality`**: Rate confidence in the semantic alignment ("high", "medium", "low"). Use "medium" when the translation is paraphrased or heavily interpretive. Use "low" when alignment is uncertain.
7.  **Ensure Complete Extraction:** You MUST include ALL transliteration lines from all identified tablet editions. Do not leave any transliteration lines out. Extract ALL tablets on every page.

**CLEANING RULES:**
1.  **Flattening:** Treat text within chunks as a continuous stream. Use spaces to separate words, no newlines (`\n`).
2.  **Exact OCR:** Keep typos, brackets `[...]`, half-brackets `⌈...⌉`, and special chars (š, ṣ, ṭ, ₄, ₆) exactly as seen.
3.  **Strip Markers:** Remove the surface indicators (`Obv.`, `Rev.`, `Lo.e.`, `Le.e.`, `Up.e.`, `U.e.`, `R.e.`) and the original margin line numbers from the final `transliteration` strings so only the actual Akkadian text remains.
4.  **Footnote markers:** Remove superscript footnote numbers from the transliteration text (e.g., remove the `⁹` from a line if it refers to a footnote). Keep only the Akkadian text.
5.  **Editorial notes:** Remove parenthetical editorial notes like `(erasure)`, `(empty line)`, `(sealing A)`, `(sealing B)` from transliteration lines — these are editorial annotations, not Akkadian text.
